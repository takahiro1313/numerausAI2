> **実行方式**：**Community Cloud にデプロイ**（学習・推論の計算が役員PCに重いため）。参加者は配布URLを開き、**アプリ内の編集セル**に書く（サーバ側 exec）。IT編（ローカル直編集）とは方式が異なる。
> **v5 の要点**：①手がかり（特徴量）を `["問い合わせ数","面談数",…]` のように**活動の名前で書く**（`X_train` のような記号は使わない）。②**モデル作成と fit を1つのセル**に統合。③特徴量を「今月の活動4種 → 来月の成約数（1ヶ月リード）」にシンプル化（12ラグ列は廃止）。

## 1. 背景・位置づけ

- 90分ハンズオンの **第2パート＝数値AI（35分）**。**数字→数字**を予測する。
- 通しテーマ②「**AIがデータからルールを学ぶ**」。IT①「人が if を書く」との対比。
- ねらい：①AIはデータから学ぶ ②入力→処理→出力はどのAIも同じ ③改善は反復。

### 1.1 一本の糸

学習の出力＝来月予測は `予測結果.csv` に保存され③まーぴーが話す。共通構造は [`../../00_共通/ハンズオン通しテーマ_一枚絵.md`](../../00_共通/ハンズオン通しテーマ_一枚絵.md)。

---

## 2. 目的・ゴール

「**数字を入れたら数字が出る**」数値AIを、参加者が **アプリ内編集セルに書いて** 動かす。

- 題材：**今月の営業活動 → 来月の成約数を予測**（1ヶ月リードのフォーキャスト）。
- **手がかり（特徴量）＝活動の名前で選んで書く**：`問い合わせ数 / 架電数 / 面談数 / 案件提案数`。何を見せるかを自分で決める。
- 答え（y）＝**翌月の成約数**。
- モデル：XGBoost 既定。`LinearRegression()` も書ける（モデル選び）。
- 副次：重要度を見ると「**架電（量）より、提案・面談が来月の成約に効く**」。手がかりから架電を外しても当たり方が変わらない＝「架電は効かない」を自分の手で確認できる。

## 4. 題材・データ（SFDCイメージのダミー時系列）

- **営業担当 × 月の時系列パネル**。今月の活動 → 来月の成約数を予測。

### 4.1 生データのスキーマ

| カラム | 例 | 役割 |
|---|---|---|
| `営業担当` | 濵田隼人 | キー（実名。固定8名＋ランダム67名） |
| `月` | 1〜12 | 時間（暦の月） |
| `問い合わせ数`/`架電数`/`面談数`/`案件提案数` | … | **今月の活動（手がかりの候補）**。架電はあえて効かない／提案が最も効く |
| `成約数` | 5 | その月の成約（→翌月にズラして答え y に） |

- 規模：**75担当 × 12ヶ月 ＝ 900行**。
- **特徴量＝今月の活動4種**（当月のみ。旧「当月・1・2ヶ月前の12列」は廃止）。**答え＝翌月の成約数**（`成約数` を1ヶ月先にズラす＝🔒前処理）。
- 成約は「先月の提案・面談」で決まるよう生成（1ヶ月リード）。固定8名の多寡は提案・面談のベース倍率(1.4/0.6)で作る。

---

## 5. 当日の進め方

配布URLを開き、🟢の編集セルだけ触る。

| Step | 内容 | やること |
|---|---|---|
| Step1 | データを見る | 手がかり＝活動／答え＝翌月成約（🔒 見るだけ・全件） |
| Step2 | 手がかりを書いて学習 | `手がかり = ["問い合わせ数", ...]` ＋ `model = ...` ＋ `model.fit(訓練データ[手がかり], 訓練データ["翌月の成約数"])` → 🎓 学習（🟢） |
| Step3 | 推論する | 担当を選び `pred = model.predict(来月予測用)` → 🔮 推論（🟢） |
| 反復 | 手がかり・数字を変える | 手がかりから架電を外す／モデルの数字を上げる → 当たり方・過学習を観察（🟢） |

- **検証は時間分割**：序盤の月で学習し、終盤の月（未来）で検証。

### 5.1 3パート連結（②→③まーぴー）

- 学習が通ると、全担当の来月予測を `data/予測結果.csv` に保存。③まーぴーがこれを読んで音声で答える。**CSV方式・実装済み**。環境差の運用は デプロイ運用メモ §3。

## 7. 機能要件（FR）

| ID | 要件 | 詳細 |
|---|---|---|
| FR-01 | データプレビュー | 担当×月の時系列を**全件**表示。手がかり(活動)／答え(翌月成約)を明示（Step1） |
| FR-02 | 手がかりを活動名で書く | 学習セルで `手がかり = ["問い合わせ数", ...]` と**活動の列名**を書く |
| FR-03 | モデル作成＋学習（Step2・1セル） | `model = XGBRegressor(...)`（or `LinearRegression()`）＋ `model.fit(訓練データ[手がかり], 訓練データ["翌月の成約数"])` → [学習] |
| FR-04 | 推論（Step3） | 担当を選び `pred = model.predict(来月予測用)` → [推論] で来月予測成約数を表示 |
| FR-05 | 過学習・手がかりの可視化 | train/val スコア対比（時間分割）。手がかりを変える／数字を上げると当たり方が変わるのが見える |
| FR-06 | 特徴量重要度 | 木モデルのとき、選んだ手がかりの重要度を棒グラフ（提案・面談が高い／架電は低い） |
| FR-07 | 編集範囲の明示（🔒/🟢） | 画面の一覧表とセル内コメントで明確化 |
| FR-08 | エラーの画面表示 | 編集コードが不正でも落ちず、例外を捕捉して画面表示 |
| FR-09 | 演習用／正解用の2系統 | 編集セル初期値が**空欄／記入済み**の2系統 |
| FR-10 | 連結CSV出力 | 学習が通ると全担当の来月予測を `data/予測結果.csv` に保存（③連結） |

### 7.1 参加者が書く対象（🟢）

1. **手がかり**：`手がかり = ["問い合わせ数", "面談数", ...]`（活動の列名）
2. **モデル＋学習**：`model = ...` ＋ `model.fit(訓練データ[手がかり], 訓練データ["翌月の成約数"])`（1セル）
3. **推論**：`pred = model.predict(来月予測用)`

→ それ以外（データ・翌月成約づくり・時間分割・スコア・グラフ・CSV保存）は 🔒。`訓練データ`/`来月予測用`/`XGBRegressor`/`LinearRegression` は名前空間で渡す。

## 10. 受入基準

- [ ] 配布URLで、手がかり（活動名）＋model＋fit を書いて [学習] すると train/val スコアと重要度が出る
- [ ] `pred = model.predict(来月予測用)` を書いて [推論] すると担当の来月予測成約数が出る
- [ ] 手がかりを変える（例：架電だけ → スコアが落ちる）／数字を上げる → 過学習 が観察できる
- [ ] 重要度で「提案・面談が効く／架電は効かない」が見える
- [ ] 学習が通ると `data/予測結果.csv` が生成され、③まーぴーが読める
- [ ] 演習用（空欄）／正解用（記入済み）の2系統が揃い、正解版はそのまま動く

## 11. 確定事項・未確定

### 11.1 確定

- **実行＝Cloud デプロイ。編集セル＋サーバ側 exec**。
- **手がかり＝活動名で書く**（`X_train` は使わない）。**model＋fit は1セル**。**推論は別セル**。
- 特徴量＝**今月の活動4種**（12ラグ廃止）。答え＝翌月の成約数（1ヶ月リード）。
- モデル＝XGBoost 既定＋線形回帰も可。過学習＝train/val対比。検証＝時間分割。
- 担当は実名（固定8名＝多寡を仕込む＋ランダム67名）。連結＝CSV方式（実装済み）。

### 11.2 未確定・要確認

1. 特徴量・数値の現実性（本番SFDCの活動定義・粒度・リードタイムをTech0/現場と擦り合わせ）。
2. リードタイム：当日は「今月→翌月（+1ヶ月）」。長い営業サイクルなら +2〜3ヶ月等。
3. 連結CSVの環境差運用（AI=Cloud／まーぴー=ローカル）：同梱サンプル or 講師が書き出して配置。
